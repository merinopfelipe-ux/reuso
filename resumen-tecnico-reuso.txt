MÓDULO DPP — FLUJO COMPLETO
----------------------------

1. empresa_admin CREA ACTIVO
   → genera código DPP-2026-XXXX-ABCD12
   → calcula hash SHA-256 encadenado con el DPP anterior (blockchain ligero)
   → sube imagen → comprime a WebP (Canvas API, cliente)

2. empleado REGISTRA CICLO
   → operación realizada, fechas, distancia transporte
   → CO2 ciclo = distancia × 0.21 kg/km (factor DEFRA)
   → CO2 evitado = suma(material.peso_kg × material.factor_co2_kg)

3. empresa_admin CALCULA MÉTRICAS FINANCIERAS
   → ingresa costos en formulario de 3 grupos colapsables
   → motor puro financiero.ts calcula: TCO, E-ROI, ICE, Inflow circular
   → narrativa en voz activa generada automáticamente para el CFO
   → PDF reporte descargable con jsPDF

4. empresa_admin SUBE DOCUMENTO (factura/foto)
   → upload a Supabase Storage privado
   → POST /api/dpp/ingesta/procesar-ia
       → Gemini 2.0 Flash lee la imagen
         (prompt específico por tipo de documento)
       → si confianza < 0.6 → fallback OpenRouter Qwen-VL
       → si hay campos con confianza < 0.7 → Groq LLaMA valida coherencia
   → usuario revisa tabla: campos, confianza % y destino DPP
   → "Acepta y pre-llena las métricas" → Tab Métricas con valores cargados

5. Patricia (consumidor) ESCANEA QR
   → /pasaporte/[codigo] — ISR revalidate 3600, sin auth
   → muestra: identidad, composición con fuentes, ciclos,
     CO2 evitado, hash parcial
   → botón "Compartir en WhatsApp" para marketing viral
   → registra visita en dpp_verificaciones (fire-and-forget)


MOTOR FINANCIERO (src/lib/calculos/financiero.ts)
--------------------------------------------------

Función pura sin efectos secundarios. 5 fórmulas:

  Métrica           | Fórmula
  ------------------|----------------------------------------------------------
  TCO               | c_adquisicion + c_operacion + c_mantenimiento
                    | + c_disposicion - v_reventa
  TCO unitario      | TCO / max(n_ciclos, 1)
  Costo evitado     | (p_virgin_usd_kg × q_circular_kg) + c_disposicion
                    | + c_impuesto_evitado
  E-ROI             | ((ahorro_operativo + costo_evitado) / inversion_ce) × 100
  ICE               | ((fp_ce - fp_lineal) / fp_lineal) × 100
  Inflow circular   | ((m_secundario + m_renovable) / m_total) × 100

13 tests unitarios Vitest — 0 fallos.


SISTEMA DE IA — 3 ROLES DISTINTOS (no intercambiables)
--------------------------------------------------------

  Proveedor    | Modelo          | Rol                        | Cuándo actúa
  -------------|-----------------|----------------------------|----------------------
  Gemini       | 2.0 Flash       | Extractor visual primario  | Siempre primero
  OpenRouter   | Qwen-VL 8B      | Extractor visual alternativo | Si Gemini falla o
               |                 |                            | confianza < 0.6
  Groq         | LLaMA 3.3 70B   | Validador coherencia DPP   | Solo si campos con
               |                 | (sin visión, solo texto)   | confianza < 0.7

Prompts dinámicos por tipo de documento:
  - factura_compra   → busca precio, precio/kg, cantidad
  - recibo_energia   → busca costo total del servicio
  - certificado_origen → busca pesos en kg, % reciclado
  - foto_objeto      → busca peso en etiqueta o placa

Parámetros: maxOutputTokens: 512, temperature: 0.
Ninguna IA calcula fórmulas — eso lo hace financiero.ts.
