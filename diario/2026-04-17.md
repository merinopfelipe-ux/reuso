# Sesión continuación — Bloque 8: Módulos comprables

## Cambios principales

### SQL
- `sql/007_modulos.sql` — tablas `modulos` y `modulos_empresas`, FK `modulo_id` en `categorias`, RLS e índices

### Tipos y schemas
- `src/types/index.ts` — añadidos `Modulo`, `ModuloEmpresa`, `ModuloConCategorias`, `ModuloConActivo`; campo `modulo_id` en `Categoria`
- `src/lib/schemas/modulo.schema.ts` — `crearModuloSchema`, `patchModuloSchema`, `patchModuloEmpresaSchema`

### APIs
- `GET/POST /api/admin/modulos` — lista con counts + crear
- `PATCH/DELETE /api/admin/modulos/[id]` — editar/toggle/eliminar
- `GET/PATCH /api/admin/empresas/[id]/modulos` — módulos con flag `activo_en_empresa` + upsert

### Páginas admin
- `/admin/modulos` — grid de cards con crear/editar/toggle + preview ícono Lucide
- `/admin/empresas/[id]` — sección "Módulos activos" con toggles por módulo
- `/admin/categorias` — selector de módulo en form + badge en lista
- `sidebar.tsx` — link "Módulos" con ícono Layers

### Calculadora (filtro)
- `dashboard/page.tsx` y `empresa/page.tsx` — filtran categorías por módulos activos de la empresa
- `usuario_libre` (sin empresa_id) → `moduloIdsActivos = null` → sin filtro, ve todas
- Empresa sin módulos → `data: null` → `[]` → EmptyState en calculadora

## Decisiones técnicas
- `modulo_id` incluido en SELECT de Supabase para respetar tipo `Categoria` actualizado en TypeScript
- Early return con `Promise.resolve({ data: null })` cuando empresa no tiene módulos — evita query innecesaria y devuelve EmptyState
