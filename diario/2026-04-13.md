# Sesión 2026-04-13 — V3.8: Ancho, roles, perfil, certificados

## Contexto de entrada

Continuación de [[2026-04-10]] (V3.7). La plataforma tenía errores de hidratación en la landing, el ancho del panel no centraba correctamente, el super_admin podía ver UI de empleado/empresa, el formulario de perfil no guardaba correctamente, y la generación de certificados PDF no funcionaba.

---

## Cambios implementados

### Hidratación: `<style>` con children string

- **Problema:** React SSR codifica comillas simples como `&#x27;`, el cliente las renderiza como `'`. Mismatch → error de hidratación en consola.
- **Archivos afectados:** `page.tsx` (landing), `header.tsx`, `sidebar.tsx`, `panel-certificados.tsx`.
- **Solución:** Cambiar `<style>{...}</style>` por `<style dangerouslySetInnerHTML={{ __html: \`...\` }} />` en todos los casos.
- Ver [[hydration-style-dangerouslySetInnerHTML]].

### Ancho correcto en panel autenticado

- **Problema:** `maxWidth + margin: 0 auto` en un flex item dentro de `flexDirection: 'column'` no funciona porque `align-items: stretch` anula el centrado.
- **Solución:** Añadir un `<div>` block-level DENTRO del `<main>` con `maxWidth: 1200, margin: '0 auto'`. Los elementos block sí respetan esta regla.
- **Archivo:** `src/components/layout-shell.tsx`
- Ver [[margin-auto-flex-vs-block]].

### Roles: separar super_admin de dashboard y empresa

- **Problema:** `(dashboard)/layout.tsx` sin protección de rol → super_admin veía UI de empleado. `(empresa)/layout.tsx` permitía super_admin explícitamente.
- **Solución:**
  - `(dashboard)/layout.tsx`: añadir `if (rol === 'super_admin') redirect('/admin')`
  - `(empresa)/layout.tsx`: reemplazar guard → `if (rol === 'super_admin') redirect('/admin')` + `if (rol !== 'empresa_admin') redirect('/dashboard')`
- Ver [[role-routing-nextjs]].

### Identidad super_admin en UI

- **`src/components/header.tsx`:** Reemplazar nombre empresa por chip "Panel de Control" cuando `rol === 'super_admin'`.
- **`src/app/(admin)/admin/page.tsx`:** Añadir chip "Super Admin · Sistema" SOBRE el saludo cálido existente. NO reemplazar la voz activa.

### Formulario de perfil: Nombres y Apellidos

- **`src/app/settings/page.tsx`:** Campos "Nombres" y "Apellidos" en la misma fila (`display: flex, gap: 12`), ambos `required`. Campo "¿Cómo quieres que te llamemos?" (apodo) como tercero.
- Después de guardar: llamar `router.refresh()` para actualizar saludo en header.
- **Decisión de diseño:** La voz activa y el tono propositivo son la base del proyecto. NUNCA reemplazar el saludo cálido con headers formales.

### Schema: apellido y apodo en profiles

- **Problema:** La tabla `profiles` no tenía columnas `apellido` ni `apodo`. El API hacía upsert con esos campos → Supabase los rechazaba → error silencioso.
- **`sql/001_schema_inicial.sql`:** Añadir `apellido text NOT NULL DEFAULT ''` y `apodo text`. Bloque de migración al final del archivo con `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`.
- **Pendiente para el usuario:** Ejecutar el bloque de migración en el SQL Editor de Supabase.

### API perfil: resiliente y correcto

- **`src/app/api/profile/route.ts`:**
  - GET: retorna `{ email: user.email }` si no hay fila en profiles (en lugar de 404).
  - PATCH: usa `upsert` con `onConflict: 'user_id'` en lugar de `update` (que falla silenciosamente si no hay fila).
  - Schema Zod: `apellido: z.string().min(1).max(100)` (obligatorio).
- Ver [[supabase-upsert-onconflict]].

### displayName: usa apellido

- **`src/lib/display-name.ts`:** Actualizado para concatenar `nombre + apellido` cuando ambos existen, o mostrar apodo si está definido.

### Certificados: corrección lógica en API

- **Problema 1:** Bloque `else` agresivo en `route.ts` devolvía 403 a empresa_admin cuyo `empresa_id` venía del perfil (no del body).
- **Problema 2:** `(dashboard)/dashboard/certificados/page.tsx` no pasaba `empresaId` al componente `PanelCertificados`.
- **Solución:** Reestructurar resolución de `empresaId` como `input.empresa_id ?? perfil?.empresa_id ?? null`, luego verificar con `if (!empresaId)`. Pasar `empresaId` desde la página server component.

### PDF: rebranding final

- **`src/lib/pdf/generar-pdf.ts`:** "reuso.bio" → "Calculadora de Reúso" / "reuso.lurdes.co".

## Decisiones técnicas

### Voz activa: intocable
El tono propositivo y el saludo cálido son la identidad del proyecto. Cualquier mejora UI debe AÑADIR elementos (chips, labels) POR ENCIMA del saludo, nunca reemplazarlo.

### margin: 0 auto falla en flex items de columna
`align-items: stretch` en un flex container con `flexDirection: 'column'` invalida `margin: 0 auto` en los hijos. La solución es siempre un div block-level intermedio. Ver [[margin-auto-flex-vs-block]].

### upsert sobre update en Supabase
Si el perfil puede no existir (usuario nuevo, fila no creada aún), siempre usar `upsert` con `onConflict`. `update` falla silenciosamente cuando no encuentra la fila. Ver [[supabase-upsert-onconflict]].
