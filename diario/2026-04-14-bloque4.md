function itemsDeDetalle(detalle): DetalleItem[] {
  // Filtra claves que empiezan con '_' (son campos meta como _descripcion_html)
  return Object.entries(detalle)
    .filter(([k, v]) => !k.startsWith('_') && typeof v === 'object')

}

## Decisiones técnicas

### Modal como panel lateral, no modal centrado
Un panel lateral (drawer desde la derecha) es mejor UX para detalles de tabla: el usuario ve el panel sin perder contexto de la tabla detrás. El overlay semitransparente con blur mantiene conciencia de dónde está.

### body overflow: hidden en el modal
Cuando el modal está abierto se previene el scroll del body principal. Se implementa via `<style dangerouslySetInnerHTML>` dentro del modal — al desmontar el componente, el estilo desaparece automáticamente. Ver [[hydration-style-dangerouslysetinnerhtml]].

### Compatibilidad legado (cantidad) vs nuevo (peso_kg)
Registros anteriores al Bloque 3 tienen `cantidad: number`. Los nuevos tienen `peso_kg: number`. El componente detecta cuál usar: `item.peso_kg != null ? '${peso} kg' : '${cantidad} u.'`

### JOIN manual en empresa/calculos
El Server Component hace la consulta de profiles por separado y construye un `Map<userId, nombre>` para enriquecer las filas. Esto es más eficiente que un JOIN en Supabase porque ya tenemos los `user_id` únicos.

### Guard de modoEmpleado en empresa/calculos
Si `empresa_admin` está en modo colaborador (`modoEmpleado = true`), esta página lo redirige al dashboard. Ver [[modo-empleado-cookie]].
